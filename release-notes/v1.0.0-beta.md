# Ogoune v1.0.0-beta

**Uptime monitoring that confirms failures before it cries wolf.**

This is the **first public release** of Ogoune. Most monitoring tools alert on
the first failed check — Ogoune verifies a failure across N consecutive checks
before it raises an incident, so every alert is real. No more 3am pages for a
2-second network blip.

> **Beta.** The public HTTP API (`/api/v1/*`) is considered stable, but this is
> an early release — expect rough edges. Please send feedback and bug reports on
> [GitHub Discussions](https://github.com/denisakp/ogoune/discussions).

---

## Highlights

**Monitoring**
- Monitor types: HTTP/HTTPS, TCP, DNS, ICMP, Keyword/content, and application
  Protocol checks.
- SSL certificate expiry alerts (J-30 / J-14 / J-7 / J-1) and domain expiry
  alerts (WHOIS), with SSL/domain metadata enrichment.
- Heartbeat / Push monitoring for cron jobs and background workers — ping via
  `GET`/`POST /ping/{slug}`; missed pings open an incident, recovery pings
  resolve it automatically.

**Alerting & incidents**
- Confirmation window (N consecutive failures) before an incident is raised.
- Flap detection, cooldown (single down-alert per incident), timed reminders,
  and component-level alert grouping.
- Full incident lifecycle (create / resolve) with rich diagnostics,
  human-readable causes, and an event timeline.

**Notifications**
- SMTP and webhooks: Slack, Discord, Google Chat, Teams, and generic HTTP.
- Channel credentials encrypted at rest (AES-256-GCM).

**Surface & operations**
- Public status page with component aggregation, 90-day uptime bars, and
  maintenance windows (one-time + cron).
- Automated monthly uptime reports delivered by email.
- Bulk import/export of resources via a YAML manifest.
- API keys (read / read_write scopes), TOTP 2FA, components, tags.

**Runtime**
- Community Edition: single binary, embedded SQLite, in-process TimingWheel
  scheduler — zero external dependencies.
- Production stack: PostgreSQL + Redis/Asynq, stateless API + external workers.

## Install

### Community Edition (recommended — zero dependencies)

```bash
docker run -d \
  --name ogoune \
  --restart unless-stopped \
  -p 8080:8080 \
  -v ogoune_data:/data \
  -e APP_SECRET_KEY=$(openssl rand -hex 32) \
  ghcr.io/denisakp/ogoune:v1.0.0-beta
```

Open **http://localhost:8080**.

`APP_SECRET_KEY` is mandatory — the app refuses to start without it.

Full setup (Docker Compose, full stack, build from source, first login,
environment reference) is in **[QUICKSTART.md](../QUICKSTART.md)**.

## Known limitations (beta)

- Enterprise Edition detection is runtime metadata only and does not yet gate
  any behaviour.
- First public release — the upgrade path across future betas is not yet
  guaranteed. Back up your data volume before upgrading.
- Report anything unexpected on
  [GitHub Discussions](https://github.com/denisakp/ogoune/discussions).
