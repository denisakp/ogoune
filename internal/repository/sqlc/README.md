# `internal/repository/sqlc/` — generated SQL bindings

Type-safe query bindings generated by [sqlc](https://sqlc.dev) from the
`.sql` files under `queries/{postgres,sqlite}/`. **This is the only ORM
in the project.** GORM was removed in spec 052 (see
[ADR 0003](../../../docs/adrs/0003-sqlc-replaces-gorm.md)).

## What's here

```text
internal/repository/sqlc/
├── pg/                 # Generated PG bindings (DO NOT EDIT)
├── sqlite/             # Generated SQLite bindings (DO NOT EDIT)
├── queries/
│   ├── postgres/*.sql  # Hand-written PG queries
│   └── sqlite/*.sql    # Hand-written SQLite queries (paired with PG)
├── dynquery/           # Dialect-aware dynamic filter builder (spec 051)
├── README.md           # This file
└── PATTERNS.md         # Canonical implementations of recurring patterns
```

## Authoring workflow — adding a new repository

End-to-end recipe for adding a `Webhook` repository. Zero GORM mentions.

### 1. Schema migration

Author paired migrations under both dialects:

```text
internal/database/migrations/postgres/0030_webhooks.up.sql
internal/database/migrations/postgres/0030_webhooks.down.sql
internal/database/migrations/sqlite/0030_webhooks.up.sql
internal/database/migrations/sqlite/0030_webhooks.down.sql
```

PG example:

```sql
CREATE TABLE IF NOT EXISTS webhooks (
    id          TEXT PRIMARY KEY,
    resource_id TEXT NOT NULL REFERENCES resources(id) ON DELETE CASCADE,
    url         TEXT NOT NULL,
    enabled     BOOLEAN NOT NULL DEFAULT TRUE,
    created_at  TIMESTAMPTZ NOT NULL,
    updated_at  TIMESTAMPTZ NOT NULL
);
CREATE INDEX IF NOT EXISTS idx_webhooks_resource_id ON webhooks(resource_id);
```

Symmetric SQLite (TEXT for timestamps, no `IF NOT EXISTS` quirks per
CLAUDE.md gotchas).

Verify the file pair + column parity:

```bash
make migrations-drift-check
```

### 2. sqlc queries

Author paired query files:

```text
internal/repository/sqlc/queries/postgres/webhook.sql
internal/repository/sqlc/queries/sqlite/webhook.sql
```

PG example:

```sql
-- name: CreateWebhook :exec
INSERT INTO webhooks (id, resource_id, url, enabled, created_at, updated_at)
VALUES ($1, $2, $3, $4, $5, $6);

-- name: FindWebhookByID :one
SELECT * FROM webhooks WHERE id = $1;

-- name: ListWebhooksByResourceID :many
SELECT * FROM webhooks WHERE resource_id = $1 ORDER BY created_at DESC;

-- name: UpdateWebhook :execrows
UPDATE webhooks SET url = $2, enabled = $3, updated_at = $4 WHERE id = $1;

-- name: DeleteWebhook :execrows
DELETE FROM webhooks WHERE id = $1;
```

Symmetric SQLite version (placeholders are `?` not `$N` — sqlc handles
the translation).

Generate:

```bash
make sqlc-generate
```

This produces:

- `internal/repository/sqlc/pg/webhook.sql.go`
- `internal/repository/sqlc/sqlite/webhook.sql.go`

Both add typed methods to `*Queries`. **DO NOT EDIT** the generated
files — every change to the `.sql` file regenerates them.

### 3. Domain model

In `internal/domain/models.go`:

```go
type Webhook struct {
    Base                       // pulls in ID, CreatedAt, UpdatedAt
    ResourceID string `json:"resource_id"`
    URL        string `json:"url"`
    Enabled    bool   `json:"enabled"`
}
```

**No `gorm:"..."` tags. No hooks.** The SQL migration is the schema
source of truth.

### 4. Port interface

In `internal/port/repository.go`:

```go
type WebhookRepository interface {
    Create(ctx context.Context, w *domain.Webhook) (*domain.Webhook, error)
    FindByID(ctx context.Context, id string) (*domain.Webhook, error)
    ListByResourceID(ctx context.Context, resourceID string) ([]*domain.Webhook, error)
    Update(ctx context.Context, w *domain.Webhook) error
    Delete(ctx context.Context, id string) error
}
```

### 5. sqlc-backed wrapper

Author `internal/repository/store/webhook_repository_sqlc.go`. Follow
the canonical shape in [`PATTERNS.md`](./PATTERNS.md) — dual-dialect
dispatch, explicit ID generation via `EnsureID()`, explicit timestamp
assignment.

Add compile-time check in `internal/repository/store/verify.go`:

```go
var _ port.WebhookRepository = (*WebhookRepositorySQLC)(nil)
```

### 6. Contract test (dual-dialect)

Author `internal/repository/store/webhook_repository_contract_test.go`:

```go
func TestWebhookRepository_Contract(t *testing.T) {
    internaltest.ForEachDialect(t, func(t *testing.T, fx *internaltest.DialectFixture) {
        repo := store.NewWebhookRepositorySQLC(fx.Runtime)
        runWebhookContract(t, repo, fx)
    })
}
```

Run:

```bash
make test-be        # SQLite only (default)
make test-be-pg     # Both dialects via testcontainer (needs Docker)
```

### 7. Fake (for handler/service unit tests)

Author `internal/repository/fake/webhook_fake.go` — pure in-memory impl
satisfying `port.WebhookRepository`. Used by handler tests that don't
need a real DB.

### 8. Wire in bootstrap

In `internal/platform/bootstrap/database.go`:

```go
app.WebhookRepo = store.NewWebhookRepositorySQLC(rt)
```

There is no flag-based dispatch — the sqlc impl is the only impl, wired
unconditionally.

### 9. Final verification

```bash
make ci-local
```

Should exit 0. CI also runs `test-be-postgres` for the dual-dialect lane.

## Gotchas

- **modernc.org/sqlite** binds `time.Time` parameters using Go's `String()`
  format, not RFC 3339. SQLite's `strftime('%s', col)` returns NULL on
  the Go format. If your SQLite query computes Unix seconds from a
  timestamp column, use `substr(col, 1, 19)` to extract the
  `YYYY-MM-DD HH:MM:SS` prefix first. See `FindMissedHeartbeatsSQLite`
  for the canonical workaround.
- **`pgx/v5`** placeholders are `$N`, not `?`. sqlc generates the right
  one automatically per dialect.
- **`gorm:"..."` struct tags** must not appear in any new code. Spec 052
  explicitly removed all 115 occurrences from `internal/domain/models.go` —
  please don't reintroduce.
